Короткое описание приложения
Мобильный цифровой гардероб: пользователь фотографирует вещи или импортирует изображения из интернета, ИИ удаляет фон, распознаёт, что это именно одежда, и собирает из нескольких фото канонизированный ассет (front/back/on-model). Далее пользователь подбирает образы и запускает виртуальную примерку на своём фото: поза/ключевые точки считаются локально на устройстве для скорости и приватности. Все исходники и результаты хранятся в облаке, контент отдается через CDN. MVP — 2D try-on; post-MVP — просмотр на 3D-аватаре.
Технологический стек
Платформы: iOS/Android (Expo + React Native).
Фронтенд: expo-camera, expo-image-picker, WebView/бридж для PICTOFiT; локально — MediaPipe BlazePose или MoveNet (TFLite).
Бэкенд: C#/.NET (Azure Functions/App Service) — загрузка, прокси к внешним ИИ-API, оркестрация пайплайна, токены/сессии, биллинг/логирование.
ИИ-сервисы:
Удаление фона/маска — Clipdrop Remove Background API.
Канонизация одежды из нескольких фото + виртуальная примерка — Reactive Reality PICTOFiT (SDK + Content Service).
Хранилище/выдача: Azure Blob Storage + Azure CDN, SAS-ссылки.
Данные: Azure SQL или Cosmos DB (метаданные вещей, ассеты, атрибуты, сеты).
Аутентификация: Azure AD B2C / JWT.
Наблюдаемость: Application Insights.
3D (post-MVP): Ready Player Me (GLB-аватары) + просмотр в RN (WebView/react-three-fiber/expo-three).


Стек, который даст быстрое и качественное MVP и нормально ляжет на C# + Expo/React Native + Azure:

1. Удаление фона / маска вещей
→ Clipdrop Remove Background API (HTTP POST, multipart). Быстро, стабильный прод-API, сразу PNG с альфой. Используй как препроцессинг при загрузке вещей. 
clipdrop.co

2. Канонизация одежды из нескольких фото + виртуальная примерка
→ Reactive Reality PICTOFiT (SDK + Content Service). Это end-to-end: из нескольких изображений одной вещи делают «умный» ассет (2D/3D) и дают VTO «на человеке». Есть SDK/Web SDK; в RN подключается через WebView/бридж. Это самый прямой путь, чтобы из нескольких фоток одной вещи получить пригодный для примерки ресурс без своей R&D-фермы. 
Reactive Reality
docs.pictofit.com

3. Поза/ключевые точки на устройстве (для скорости и приватности)
→ MediaPipe BlazePose или MoveNet (TFLite) — запускай локально в приложении, отдавай на бэкенд только нужные карты/метаданные. 
blog.tensorflow.org

4. Инфра и хранение
→ Azure Blob Storage + CDN для исходников и результатов; бэкенд-прокси на Azure Functions/App Service для вызова Clipdrop/PICTOFiT. (Стандартно, без экзотики.)

5. 3D-аватар (после MVP)
→ Ready Player Me (есть гайд под React Native; аватары в GLB). Подключается через WebView/SDK, хорошо ложится в мобильный стек. 
docs.readyplayer.me


Если по каким-то причинам нельзя SaaS для примерки, альтернатива: self-host TryOnDiffusion/HR-VITON на Azure ML/Container Apps с GPU, но это дольше по интеграции и MLOps (береги время — для MVP бери PICTOFiT).
